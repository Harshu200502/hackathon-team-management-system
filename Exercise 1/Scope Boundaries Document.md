## Scope Boundaries Document

This document defines the scope boundaries for the **Hackathon Team Management System** by clearly specifying what is included and excluded from the project. It helps manage expectations and prevents scope creep during development.

---

### In-Scope Boundary

The system **will include**:
- User registration and authentication  
- Team creation and member management  
- Role assignment within teams  
- Viewing and managing team information  
- Basic user profile management  
- Centralized platform for team coordination  
- Project documentation and UML modeling using a CASE tool  

---

### Out-of-Scope Boundary

The system **will not include**:
- Payment or financial transaction features  
- Real-time chat or video conferencing  
- Mobile application development  
- Integration with third-party services or APIs  
- Advanced analytics or reporting dashboards  
- AI-based recommendations or automation  
- Multi-event or large-scale event management
