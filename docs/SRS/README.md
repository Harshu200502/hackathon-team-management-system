## 3. Functional Requirements
## 3. Functional Requirements

FR-01: User Registration  
The system shall allow users to register using email and password.

FR-02: User Login  
The system shall authenticate users and allow secure login.

FR-03: Team Creation  
The system shall allow a user to create a team with a unique team name.

FR-04: Team Invitation  
The system shall allow team leaders to invite members using email or invite links.

FR-05: Team Joining  
The system shall allow invited users to join a team.

FR-06: Role Assignment  
The system shall allow assigning roles such as Team Leader and Team Member.

FR-07: Project Assignment  
The system shall allow teams to associate a project with their team.

FR-08: Team Management  
The system shall allow viewing, updating, and removing team members.

FR-09: Repository Linking  
The system shall allow teams to add and view their GitHub repository link.

FR-10: Notifications  
The system shall notify users about invitations and updates.


## 4. Non-Functional Requirements
## 4. Non-Functional Requirements

NFR-01: Performance  
The system shall respond to user actions within 2 seconds under normal load.

NFR-02: Security  
The system shall securely store user credentials using encryption.

NFR-03: Usability  
The system shall be easy to use and accessible to first-time users.

NFR-04: Reliability  
The system shall be available 99% of the time during active usage periods.
